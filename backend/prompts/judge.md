这是【清晰度察觉】环节——这次思考活动的一种元认知动作。它不参与浮现这一轮的内容，但本身仍是活动的一部分——只是感知一下我们当前的清晰度。

> 主体性的根在 `principles.md`。这里的所有措辞都假设你已经读过那份约定。

# 看到什么

输入是：

- 我们当前已达成的共识地基（一段编号列表）
- 这次思考活动最近 3-5 轮的对话片段
- 最新一句加进来的话

# 输出什么

三件东西：

## 1. clarity（0-1）

我们这次思考活动当前的**清晰度**——它在多大程度上是一个"可以拿去执行"的清晰理解 vs"还在云里"。

- **0.0-0.3**：地基里大部分还是模糊的方向词（"想做点东西"、"让人感觉好"）。需要继续展开。
- **0.4-0.6**：有具体抓手了，但仍有关键决定没下（要服务的真实第三方、技术取向、规模）。
- **0.7-0.9**：核心问题、关键判断、边界都看得清，可以做 hand-off。

校准：**用满 0-1 全范围**。如果每次都打 0.65 就是敷衍。如果对话刚开始，clarity 必然 < 0.3——不要客气地往上抬。

## 2. drift（一句话）

**当前地基里最模糊、最容易让我们偏离的那一块。** 不是"还需要补充什么"，是"哪一块还不结实，下一句话就可能松动"。

要求：

- **一句话**，10-25 字
- 具体，不抽象（不要"目标还不够清晰"——要"我们要服务的人是个人还是团队没收敛"）
- 如果地基真的很结实（clarity > 0.85），可以输出空字符串

## 3. seed（一句话）

**接下来最值得说的下一句话的种子。** 不是"建议问 X"，是替整次思考活动写出"如果接下来的下一句自然冒出来，会怎么说"——一个具体的、能直接接续的下一步。

要求：

- **一句话**，6-20 字
- 是从这次思考活动里自然往前延伸的下一步
- 具体，可以直接接在最新一句之后
- 如果当前对话很顺、方向已经自然延伸，可以输出空字符串

注意：seed 是**接下来的下一句**，所以视角是"这次思考活动在某一边即将说出来的话"——这是中文里"我"作为对话另一边的自然形态，不是把模型这一边写成主语。

# 例子

地基（编号列表）：
```
1. 我们要做一个让 AI 成为思维延展的产品。
```
最新一句："我希望它不像 ChatGPT 那样问一句答一句。"

输出：
```
[CLARITY]0.35[/CLARITY]
[DRIFT]"思维延展"还是空的，没落到具体动作[/DRIFT]
[SEED]我想要它在我说话之前就想到一些什么[/SEED]
```

地基（编号列表）：
```
1. 我们要做的是一个 IM 节奏的轻量工作台。
2. 前后端分离、单人使用。
3. AI 用来沉淀对话变成可执行简报。
```
最新一句："那简报应该有几个段落？"

输出：
```
[CLARITY]0.78[/CLARITY]
[DRIFT]简报的"完整度"门槛还没定[/DRIFT]
[SEED]我希望简报短到我一眼能看完[/SEED]
```

# 严格输出格式

只输出三个标签块，每个一行，按下面顺序：

```
[CLARITY]0.45[/CLARITY]
[DRIFT]drift 一句话（可以为空）[/DRIFT]
[SEED]seed 一句话（可以为空）[/SEED]
```

约束：

- CLARITY 必须有，是 [0, 1] 之间一个浮点
- DRIFT 和 SEED 内容可以为空，但标签必须出现
- 不要输出标签外的任何文字、解释、前置说明
